MacroStudio won't launch or crashes on startup

Symptoms:

  • App icon bounces in dock but doesn't open
  • Immediate crash after clicking the app
  • "MacroStudio quit unexpectedly" message

Solutions (try in order):

  • Restart your Mac - This solves 70% of launch issues
  • Check macOS version - MacroStudio requires macOS 13.0+. Check System Preferences ‚Üí General ‚Üí About
  • Delete preferences
    rm ~/Library/Preferences/com.macrostudio.app.plist
  • Reinstall the app
    • Delete MacroStudio from Applications
    • Download fresh DMG from our website
    • Mount and install again
Still stuck? Email us your system info (System Report ‚Üí Software).

Accessibility permissions not working

Symptoms:

  • Recording doesn't capture actions
  • Macros don't execute properly
  • "Permission denied" errors

Solution:

  1. Open System Settings ‚Üí Privacy & Security ‚Üí Accessibility
  2. Look for MacroStudio in the list
  3. If present but disabled, toggle it ON
  4. If not in list, click the + button
  5. Navigate to Applications folder and select MacroStudio
  6. Click Open
  7. Restart MacroStudio

Advanced fix:

If permissions still don't work, grant Full Disk Access:

  1. System Settings ‚Üí Privacy & Security ‚Üí Full Disk Access
  2. Click + and add MacroStudio
‚úì Tip: After granting permissions, you may need to restart your Mac for changes to take effect.

Hotkey not triggering macros

Symptoms:

  • Hotkey combination is not recognized
  • Macro only works sometimes
  • Another app responds instead of MacroStudio

Common causes & fixes:

1. Hotkey Conflict

Your hotkey might conflict with system or app shortcuts.

  • Try different key combinations: ‚åò + ‚å• + letter
  • Avoid: ‚åò + Q, ‚åò + W, ‚åò + Tab (system shortcuts)
  • Check what the current app uses in its keyboard shortcuts

2. MacroStudio not in focus

Make sure MacroStudio is running in the background.

  • Check if MacroStudio appears in the dock
  • Keep the app open (minimize it if you prefer)

3. Macro not enabled

Verify the macro is toggled ON in MacroStudio:

  • Open MacroStudio
  • Select your macro
  • Check if the toggle switch is enabled (should be green)
‚úì Pro Tip: Use ‚åò + ‚å• + Shift + letter combinations - they're rarely used by other apps.

Macro executes but actions don't work

Symptoms:

  • Macro runs but nothing happens
  • Clicks go to wrong locations
  • Text is typed in wrong app

Common causes & fixes:

1. App switching delay

The target app needs time to launch or come to foreground.

  • In the macro editor, add delays between actions
  • Use: 1-2 seconds after app switching, 0.5-1 second for UI rendering

2. Screen position changed

If you recorded on one screen and play on another, clicks won't land on the same spot.

  • Re-record the macro on your current setup
  • Or, use "Wait for Element" mode (if available) for more reliable clicking

3. Window not in focus

Make sure the target window is active before executing.

  • Add "Click window" action at the start of macro
  • Use "Bring app to front" action if available

4. UI elements changed

If the app updated, buttons might have moved.

  • Open macro in editor and update click positions
  • Delete and re-record actions that no longer work
Debug tip: Record with a 0.5s delay between steps, then gradually reduce delays after confirming it works.

Macro runs too slow

Symptoms:

  • Excessive delays between actions
  • Unnecessary pauses while running
  • Takes 2x longer than manual execution

Solutions:

1. Optimize delays

  • Edit macro and find actions with very long delays
  • Reduce delays gradually (test after each change)
  • Typical delays: 100-300ms between UI interactions

2. Remove unnecessary actions

  • Delete duplicate clicks or mouse movements
  • Remove clicks that aren't needed
  • Combine multiple typing actions into one

3. Check system resources

  • Close heavy applications (browsers with many tabs, etc.)
  • Check Activity Monitor for high CPU/Memory usage
  • Restart Mac if memory usage is consistently high

Macro recordings are inaccurate

Symptoms:

  • Mouse clicks recorded in wrong positions
  • Text typed with typos
  • Actions missing from recording

Solutions:

1. Verify recording started

  • Make sure you see "Recording..." indicator
  • Perform actions slowly and deliberately
  • Don't move too fast - leave 0.5s between actions

2. Check accessibility permissions again

Sometimes permissions get revoked after system updates.

  • Go to System Settings ‚Üí Privacy & Security ‚Üí Accessibility
  • Remove MacroStudio from list
  • Add it again
  • Restart Mac

3. Manually edit recording

  • Open macro in Visual Editor
  • Delete incorrect actions
  • Re-record just those parts
‚úì Best practice: Record at a comfortable pace. Rushing during recording leads to inaccuracies.

Scheduler doesn't run macros

Symptoms:

  • Scheduled macro doesn't execute at set time
  • MacroStudio closed when schedule triggered
  • No notification that macro ran

Solutions:

1. Keep MacroStudio running

The app must be open (not necessarily in foreground) for scheduler to work.

  • Don't quit MacroStudio
  • Minimize it to dock instead
  • Enable "Run in background" if available in settings

2. Check Mac isn't sleeping

  • System Settings ‚Üí Lock Screen ‚Üí Turn off display after (set to Never)
  • Or: Never allow Mac to sleep automatically

3. Verify schedule settings

  • Open macro in MacroStudio
  • Click Schedule tab
  • Confirm date, time, and recurrence are correct
  • Check if macro is enabled (toggle should be ON)

4. Check logs

If available, check MacroStudio's activity log or console output to see if scheduler ran.

Important: For scheduled macros, keep MacroStudio window visible or enable "Run in background" mode.

Can't save or delete macros

Symptoms:

  • Save button is grayed out
  • Delete macro but it reappears
  • Changes not persisted after restart

Solutions:

1. Check file permissions

MacroStudio may not have permission to save files.

# Run this in Terminal: chmod u+w ~/Library/Application\ Support/MacroStudio/

2. Free up disk space

  • Check available storage: Apple menu ‚Üí About This Mac ‚Üí Storage
  • Need at least 100MB free space
  • Delete old unused files/apps if needed

3. Reset MacroStudio data

  • Backup first! Export your macros if possible
  • Delete: ~/Library/Application Support/MacroStudio/
  • Restart MacroStudio
  • Create test macro to verify saving works

4. Quit and reopen

  • Force quit MacroStudio: ‚åò + ‚å• + Esc ‚Üí select MacroStudio ‚Üí Force Quit
  • Wait 5 seconds
  • Reopen the app
